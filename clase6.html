Idea central de la clase

Una IA no adivina. Una IA usa datos para tomar decisiones. Si los datos son útiles, la solución puede funcionar mejor. Si los datos son basura, sensibles o incompletos, la IA puede responder mal.

Regla de oro

Un dato sirve si ayuda directamente a tomar la decisión que queremos que tome la IA.

Problema real → Decisión IA → Datos útiles → Dataset → Modelo → Resultado

🎖 Insignia desbloqueada: Arquitecto de Datos IA

Completaste la misión. Ya sabes identificar datos útiles para construir un dataset de IA.

Nivel 1

Primero entiende los tipos de datos

Antes de construir un dataset, debes aprender a reconocer qué tipo de información tienes enfrente.

✅

Dato útil

Ayuda directamente a tomar la decisión que queremos que tome la IA.

Ejemplo: Para recomendar café, el presupuesto sí sirve.
🗑️

Dato basura

No aporta valor para la decisión. Puede confundir, estorbar o meter ruido.

Ejemplo: El color de mochila no sirve para recomendar una bebida.
🔒

Dato sensible

Puede invadir privacidad o ser innecesario. Solo debe pedirse si está justificado.

Ejemplo: CURP o dirección no hacen falta para recomendar un platillo.
❓

Dato faltante

Es información necesaria que no fue capturada. La IA debería pedirla antes de decidir.

Ejemplo: Si falta presupuesto, conviene preguntar antes de recomendar.

Ahora define qué resultado debe producir la IA

Esta parte sirve para aclarar qué esperamos que responda la IA. Si no sabemos qué debe producir, tampoco podremos saber qué datos necesita.

¿Para qué sirve esta decisión?

La decisión IA define la salida esperada del sistema. Después usaremos esta frase para decidir qué columnas tendrá el dataset.

Ejemplo: si la IA debe recomendar una bebida, necesitamos datos como clima, gusto y presupuesto. Pero si debe detectar un riesgo, necesitaremos señales, síntomas o indicadores.

Elige una opción para recibir explicación y una frase guía.
Esto se convertirá después en:
  • Variable objetivo: lo que la IA debe responder.
  • Variables de entrada: los datos que necesita para responder bien.
  • Datos basura: información que no ayuda a esa decisión.
Nivel 3

Aprende entrada vs objetivo

Este es el concepto más importante para construir datasets de IA.

📥

Variables de entrada

🏷️

Nivel 6

Auditoría de calidad

Un dataset puede verse bonito y aun así estar mal. Aquí aprenderás a detectar problemas.

❓

Dato faltante

Falta una variable importante, como presupuesto o síntoma.

Solución:

Preguntar el dato o marcarlo como desconocido.

🔁

Dato inconsistente

El mismo valor aparece escrito de formas distintas: frappe, frappé, frape.

Solución:

Normalizar categorías.

⚖️

Sesgo

Los datos no representan toda la realidad, solo una parte.

Solución:

Recolectar ejemplos variados.

🔒

Privacidad

Se piden datos personales que no son necesarios.

Solución:

Aplicar minimización de datos.
