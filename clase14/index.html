Misión 1

Qué aprende una red neuronal

Una red neuronal aprende una relación entre entradas y salidas. No le damos una lista rígida de reglas: le damos ejemplos para que ajuste valores internos llamados pesos.

📥

Entradas

Datos del caso: temperatura, hábitos, señales del usuario o variables del proyecto.

🧠

Pesos

Números internos que indican la importancia de cada señal.

📤

Salida

La predicción o respuesta que el modelo propone.

📉

Error

La distancia entre la predicción y el resultado correcto.

La red no “entiende” como una persona. Aprende a ajustar números para que sus predicciones se acerquen más a los ejemplos correctos.
Misión 2

La neurona artificial

Una neurona artificial recibe entradas, multiplica cada entrada por un peso, combina la información y produce una salida.

entrada × peso → suma → activación → salida

Si una señal ayuda a predecir mejor, su peso puede aumentar. Si confunde al modelo, su peso puede disminuir.

Comparación

Reglas programadas vs red neuronal

Programación tradicional

La persona escribe reglas explícitas: si pasa A, entonces haz B.

Red neuronal

La persona da ejemplos. El modelo ajusta pesos para aproximarse al resultado correcto.

Pasamos de escribir todas las reglas a entrenar un modelo para que encuentre patrones.

Misión 4

Cómo entrena una red neuronal

Sección clave

Entrenar significa repetir un ciclo: el modelo predice, se compara con la respuesta correcta, calcula error y ajusta pesos para equivocarse menos la próxima vez.

Simulación

Observa cómo bajan el error y el loss

En esta simulación veremos una versión simplificada de entrenamiento. El modelo recibe un caso, hace una predicción, compara esa predicción con la salida esperada y ajusta su peso para reducir el error. No necesitas memorizar fórmulas: observa cómo el modelo se equivoca menos con cada época.

¿Qué está pasando aquí?

Estamos entrenando una neurona muy simple. Le damos una entrada numérica, como el nivel de calor, y le pedimos que produzca una salida esperada, como la probabilidad de aceptación. Al inicio el peso puede no ser correcto, por eso la predicción se equivoca. Después de cada época, el modelo calcula el error y ajusta el peso para acercarse más a la salida esperada.

Entrenar significa repetir este ciclo: predecir, comparar, medir error y ajustar.

TensorFlow

TensorFlow es una biblioteca de aprendizaje automático. Nos permite construir modelos, entrenarlos con ejemplos y pedirles predicciones. En esta página no usamos TensorFlow real: hacemos una versión visual simplificada. En Colab sí veremos código con TensorFlow/Keras.

Keras

Keras es una forma sencilla de construir redes neuronales dentro de TensorFlow. Nos permite crear modelos por capas usando menos código. Sequential es un modelo ordenado por capas y Dense es una capa donde las entradas se conectan con neuronas.

Época

Una época es una vuelta de entrenamiento. Si entrenamos 1 época, el modelo ajusta una vez; si entrenamos 20, tiene 20 oportunidades para corregirse. Más épocas no siempre significa mejor.

Guía rápida de la simulación

Calor del caso

La escala de 1 a 10 convierte una idea del mundo real en un número que el modelo puede procesar. Una red neuronal no entiende directamente la palabra “calor”; por eso la transformamos en un valor numérico.

🌡️ 1 = poco calor🌡️ 5 = calor medio🌡️ 10 = mucho calor
1–3 = clima fresco · 4–6 = templado · 7–10 = caluroso
90%

Salida esperada

Es lo que queremos que el modelo aprenda a predecir. Se maneja como porcentaje porque representa una probabilidad o nivel esperado: 80% indica alta probabilidad de aceptación; 20% indica baja probabilidad.

Funciona como la respuesta correcta durante el entrenamiento.
Tip sobre épocas: si son pocas, el modelo puede no aprender suficiente; si son demasiadas, puede memorizar demasiado. Aquí las usamos para observar cómo baja el error.

Error vs loss

Error: diferencia directa entre lo que predijo y lo que debía predecir. Si el modelo predice 60% y la salida esperada era 80%, el error es 20 puntos porcentuales.

Loss: forma de medir ese error para guiar el entrenamiento. El modelo intenta reducirlo con cada ajuste.

El error nos dice cuánto se equivocó en este intento. El loss nos ayuda a seguir la evolución del aprendizaje.
