Sistemas Expertos

Cómo representar conocimiento de especialistas mediante reglas claras y explicables.

ReglasSI / ENTONCESDiagnósticoExplicación

Definición sencilla

Un sistema experto intenta reproducir el razonamiento de un especialista mediante una base de conocimiento y un motor de inferencia. Es útil cuando el conocimiento puede escribirse como reglas.

Regla típica

SI condición o conjunto de condiciones.
ENTONCES conclusión, recomendación o acción.

Explicación técnica entendible

Primero se recopilan reglas de especialistas. Después el sistema recibe hechos del caso actual, compara esos hechos contra las reglas y llega a una conclusión. La ventaja es que puede mostrar por qué decidió algo.

Componentes

ElementoFunción
Base de conocimientoReglas y hechos.
Motor de inferenciaAplica reglas al caso.
InterfazRecibe datos del usuario.
Módulo de explicaciónJustifica la conclusión.

Diferencia con Machine Learning (ML)

Un sistema experto usa reglas escritas por humanos, explica fácilmente sus decisiones y no necesita grandes datos históricos. Machine Learning (ML) aprende patrones desde datos, puede encontrar relaciones difíciles, pero necesita ejemplos suficientes.

Ejemplo desarrollado: diagnóstico de fallas

Regla 1: SI la computadora no enciende Y el cargador no muestra luz, ENTONCES revisar fuente de energía.

Regla 2: SI la computadora enciende Y no da video, ENTONCES revisar memoria RAM o monitor.

Salida: recomendación con explicación.

1. Recibir síntomas.
2. Comparar con reglas conocidas.
3. Elegir la conclusión más compatible.
4. Mostrar qué regla se activó.

Casos de uso

Diagnóstico técnico, soporte, clasificación normativa, asesoría fiscal simple y protocolos médicos acotados.

Riesgos

Reglas incompletas, contradictorias, desactualizadas o incapaces de cubrir casos nuevos.

Cuándo conviene

Cuando especialistas pueden expresar el conocimiento como reglas claras y auditables.

3 ideas para recordar

1 Los sistemas expertos usan reglas explícitas.

2 Son útiles cuando se requiere explicar la decisión.

3 Deben actualizarse cuando cambia el conocimiento del dominio.
