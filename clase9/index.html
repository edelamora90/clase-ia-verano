Misión 1

Cargar dataset y auditar si está listo para agente

Qué es

Un dataset usable es una tabla suficientemente ordenada, con columnas entendibles, registros coherentes y una posible salida que el agente pueda aprender o usar.

Para qué sirve

Sirve para evitar diseñar un agente sobre datos que todavía no sostienen una decisión clara.

Aplicación al agente

Antes de pensar en reglas o predicciones, el agente necesita entradas confiables y una salida posible.

Ejemplo

Un dataset de cafetería con clima, presupuesto y recomendacion puede alimentar un agente recomendador.

Misión 2 · Definir el problema real del agente

Convierte tu dataset en una decisión inteligente

Un agente no se crea solo porque tenemos una tabla. Se diseña porque hay una decisión real que queremos apoyar, automatizar o explicar con datos.

Concepto clave

Problema de decisión

Un problema de decisión es una situación donde un agente debe elegir una respuesta útil con base en datos.

La respuesta puede ser una recomendación, clasificación, alerta, diagnóstico, acción sugerida, prioridad o categoría.

Para qué sirveConvierte “tengo un dataset” en “mi agente usará este dataset para tomar esta decisión”.

Misión 3

Elegir columna objetivo y variables de entrada

Columna objetivo

Es la salida que el agente intentará producir: riesgo, diagnostico, categoria, accion o recomendacion.

Variables de entrada

Son los datos que el agente recibe para decidir. Deben explicar la salida, no identificar personas sin aportar valor.

Aplicación al agente

Las entradas serán sensores de información; la columna objetivo será la respuesta esperada.

Ejemplo

Entradas: asistencia, promedio, entregas. Objetivo: nivel_riesgo.

Entradas→Agente→Objetivo
Misión 4

Validar si la columna objetivo sirve

Clases

Son los valores posibles de la columna objetivo. Por ejemplo: bajo, medio, alto.

Desbalance

Ocurre cuando una clase domina demasiado. El agente podría aprender a responder casi siempre lo mismo.

Valores únicos o vacíos

Demasiados valores únicos pueden indicar que elegiste un identificador. Vacíos en el objetivo rompen la decisión esperada.

Ejemplo

Si diagnostico tiene 95% “normal”, el agente necesita más casos variados o una decisión distinta.

Misión 5

Crear hipótesis de decisión

Qué es

Una hipótesis basada en datos es una sospecha razonable sobre cómo ciertas entradas se relacionan con la salida.

Para qué sirve

Sirve para transformar observaciones del dataset en ideas que luego podrán convertirse en reglas o modelos.

Aplicación al agente

El agente usará estas relaciones para explicar por qué una decisión tiene sentido.

Ejemplo

Cuando asistencia es baja y entregas es bajo, suele aparecer riesgo = alto.

Misión 6

Diseñar reglas candidatas SI/ENTONCES

Condición

Es la parte SI: valores observados en las variables de entrada.

Acción

Es la parte ENTONCES: recomendación, alerta, clasificación o diagnóstico.

Regla candidata

No es definitiva. Es una regla propuesta desde evidencia del dataset para probarla después.

Ejemplo

SI presupuesto = bajo y hambre = alta, ENTONCES recomendar combo escolar.

SI[condiciones observadas]ENTONCES[decisión explicada]
Misión 7

Definir PEAS

P

Performance

Cómo medirás si el agente funciona: precisión, satisfacción, rapidez o utilidad.

E

Environment

El entorno donde actúa: escuela, tienda, clínica, soporte, inventario o app.

A

Actuators

Lo que puede hacer: mostrar recomendación, alerta, prioridad, etiqueta o explicación.

S

Sensors

Los datos que recibe: columnas del dataset, formularios, historial o mediciones.

PEAS sirve para que el agente quede definido como sistema, no solo como una tabla con predicciones.
