Redes Neuronales y Machine Learning

Cómo una Inteligencia Artificial (IA) aprende patrones a partir de datos para clasificar, predecir o recomendar.

Datos históricosPredicciónClasificaciónPatrones

Definición sencilla

Machine Learning (ML), o aprendizaje automático, es una forma de Inteligencia Artificial (IA) donde el sistema aprende patrones desde ejemplos. Una red neuronal es un modelo de Machine Learning (ML) inspirado de manera simplificada en conexiones entre neuronas: recibe datos, ajusta pesos internos y produce una salida.

Jerarquía

IA

Campo general: sistemas que realizan tareas inteligentes.

Machine Learning

Subárea donde el sistema aprende con datos.

Red neuronal

Tipo de modelo de Machine Learning (ML) útil para patrones complejos.

Explicación técnica entendible

En programación tradicional una persona escribe reglas: “si pasa esto, haz aquello”. En Machine Learning (ML) se entregan ejemplos con entradas y resultados esperados; el modelo ajusta parámetros para aproximarse al resultado correcto. Después, cuando llega un caso nuevo, calcula una predicción usando lo aprendido.

Diferencia con otros conceptos

ConceptoQué es
IAÁrea completa de sistemas inteligentes.
Machine Learning (ML)Método que aprende desde datos.
Red neuronalModelo que combina capas de cálculos para detectar patrones.

Programa tradicional vs Machine Learning (ML)

Tradicional

Reglas + datos → resultado.

Machine Learning

Datos + resultados conocidos → modelo.

Uso del modelo

Nuevo caso → predicción.

Ejemplo desarrollado: alumnos en riesgo académico

Datos: asistencia, promedio, tareas entregadas, retardos y participación.

Salida: riesgo bajo, medio o alto.

Acción: tutoría, seguimiento o alerta temprana.

1. Reunir historial de alumnos.
2. Etiquetar casos anteriores según resultado académico.
3. Entrenar el modelo con esos ejemplos.
4. Evaluar si acierta en datos no vistos.
5. Usar la predicción como apoyo, no como sentencia.

Componentes

Variables de entrada, etiquetas, modelo, entrenamiento, evaluación y predicción.

Casos de uso

Riesgo académico, recomendadores, detección de fraude, demanda de ventas y clasificación de correos.

Cuándo conviene

Cuando hay datos históricos, se busca predecir o clasificar y las reglas manuales serían difíciles.

Riesgos y limitaciones

Datos incompletosSesgo históricoPrivacidadInterpretar predicción como sentenciaResultados difíciles de explicar
