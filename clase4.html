Clase 4: IA Aplicada a Problemas Reales

Cómo elegir el área de Inteligencia Artificial adecuada según el problema, los datos disponibles y la salida esperada.

Semana 1 Jueves 10:00 - 12:00 Duración: 2 horas Evidencia en PDF

1. ¿Qué significa aplicar IA?

Aplicar IA no empieza por escoger una técnica de moda. Empieza por entender qué problema real se quiere resolver, qué datos existen, qué salida será útil y qué acción tomará una persona o sistema después.

Problema→ Datos→ Técnica→ Salida→ Acción
!

Mensaje clave Sin datos claros y sin una salida útil, no hay una verdadera solución de IA.

Ejemplo: alumnos en riesgo

ProblemaIdentificar alumnos en riesgo académico.
DatosAsistencia, tareas, calificaciones y participación.
ÁreaRedes neuronales / aprendizaje automático.
SalidaRiesgo bajo, medio o alto.
AcciónTutoría o seguimiento académico.

4. Guía rápida por estrategia

Robótica

  • Usa sensores.
  • Ejecuta acciones físicas.
  • Evalúa seguridad.

Redes / Machine Learning (ML)

  • Usa datos históricos.
  • Predice o recomienda.
  • Revisa sesgos.

Visión artificial

  • Usa imágenes.
  • Detecta objetos.
  • Cuida privacidad.

Lógica difusa

  • Usa grados.
  • Controla intensidad.
  • Calibra rangos.

Procesamiento de Lenguaje Natural (PLN)

  • Usa lenguaje.
  • Clasifica o responde.
  • Valida contexto.

Sistemas expertos

  • Usa reglas.
  • Explica decisiones.
  • Mantén reglas actualizadas.

1
Actividad 1

Diagnóstico de problemas

Objetivo: identificar qué herramienta de Inteligencia Artificial (IA) conviene según el tipo de datos, la salida esperada y el riesgo del caso.

Qué deben realizar

  1. Elijan 3 problemas del “Tablero de diagnóstico”.
  2. Para cada problema, escriban qué datos recibe el sistema.
  3. Seleccionen el área principal de IA: robótica, Machine Learning (ML), visión artificial, lógica difusa, Procesamiento de Lenguaje Natural (PLN) o sistemas expertos.
  4. Definan la salida esperada: alerta, recomendación, clasificación, movimiento, respuesta o nivel.
  5. Agreguen un riesgo realista y una justificación breve.
Está bien si...

La herramienta elegida coincide con el tipo de dato principal y la justificación explica por qué no eligieron otra.

Actividad 2

Diseño de solución con IA

Objetivo: proponer una solución de IA para un problema real del entorno escolar, familiar o comunitario.

Qué deben entregar

  1. Nombre de la solución.
  2. Problema que resuelve y usuarios beneficiados.
  3. Datos de entrada y cómo se obtendrían.
  4. Área principal de IA elegida.
  5. Proceso: qué hace el sistema paso a paso.
  6. Salida esperada y acción que tomaría una persona.
  7. Riesgo, limitación y forma de evaluar si funciona.
Ejemplo de resultado esperado

Alerta Temprana Escolar

Problema: algunos alumnos bajan rendimiento antes de pedir ayuda.

Usuarios: tutores, docentes y coordinación académica.

Datos: asistencia, tareas, calificaciones y retardos.

Área: Machine Learning (ML).

Proceso: compara patrones históricos y estima nivel de riesgo.

Salida: riesgo bajo, medio o alto con recomendación de tutoría.

Evaluación: revisar si las alertas ayudan a intervenir antes de reprobar.

Está bien si...

La solución no intenta “usar IA por usar IA”, sino resolver un problema concreto con datos claros y una salida útil.

Tarea 4

Análisis de aplicación real

Objetivo: analizar una aplicación existente e identificar qué problema resuelve, qué datos usa y qué área de IA participa.

Qué debe incluir el PDF

  1. Nombre de la aplicación elegida.
  2. Problema real que resuelve.
  3. Usuarios o contexto donde se usa.
  4. Datos de entrada.
  5. Área principal de IA y áreas secundarias si aplica.
  6. Salida que entrega al usuario.
  7. Beneficio principal.
  8. Riesgo o limitación.
  9. Reflexión: qué decisión humana mejora o complica.
Ejemplo de resultado esperado

Google Maps

Problema: encontrar una ruta eficiente entre dos lugares.

Datos: ubicación, mapas, tráfico, accidentes y tiempos históricos.

Área principal: búsqueda en espacios de estados y Machine Learning (ML) para estimar tiempos.

Salida: rutas sugeridas, tiempo estimado y alertas.

Beneficio: ayuda a tomar mejores decisiones de traslado.

Riesgo: datos desactualizados o rutas poco adecuadas para peatones.

Reflexión: la IA recomienda, pero el usuario debe valorar seguridad y contexto.

Está bien si...

El análisis distingue entre datos, técnica, salida y riesgo, y no se queda solo en describir “qué hace la app”.

Si solo recuerdas 3 cosas de esta clase...

1 La IA se elige a partir del problema, no de la moda.

2 Cada área responde a un tipo diferente de datos y decisión.

3 Una buena solución combina problema, datos, técnica, salida y evaluación.

Idea final La mejor IA no es la más llamativa, sino la que resuelve mejor un problema real.
